# Cast-R1-TS 训练稳定化修改方案（v1.5）

## 1. 目标与约束

### 1.1 总体目标
本方案的目标是：

1. **让模型先学稳**：SFT 后能够稳定遵循三阶段协议，RL 前期不会大量因为协议错误、解析失败而拿不到有效学习信号。
2. **让模型逐步学好**：先学会 route，再学会 refine，最终使 `orig_mse_mean` 相比当前基线稳定下降。
3. **尽可能对齐论文**：保留 Cast-R1 的核心三阶段语义：
   - Turn 1：diagnostic / evidence gathering
   - Turn 2：routing / model invocation
   - Turn 3：reflection / refinement / final answer
4. **先排查数据问题，再调训练**：如果 teacher 标签、route 标签、refine 样本本身有噪声或失衡，那么后续再调 KL、reward、entropy 都只能缓解，不能根治。

### 1.2 不采用的方案
当前阶段**不直接采用 full v2 两步离线 rerank 重构**，原因如下：

- 会明显偏离 Cast-R1 原论文的 tool-augmented multi-turn workflow；
- 会把“当前实现到底哪里坏了”的诊断信息一起抹掉；
- 工程改动过大，无法快速判断问题来自数据、prompt、协议还是 RL 本身。

因此本方案采用 **v1.5 稳定化路线**：

- 保留 3-turn 框架；
- 修复 Turn 3 训练/运行不一致；
- 去掉 routing 泄漏式脚手架；
- 降低 RL 保守性；
- 强化高质量 route/refine 训练信号；
- 在必要时再对 Turn 3 做最小结构增强。

---

## 2. 成功判据（最终验收目标）

### 2.1 协议稳定性目标
- `invalid_decision_shape_ratio < 1%`
- `format_failure_ratio < 2%`
- `reward_negative_one_ratio < 5%`
- Turn 3 parser 成功率 > 99.5%

### 2.2 学习有效性目标
- SFT 后高置信 route 样本上的 `route agreement >= 70%`
- RL 后 `selected_model_distribution` 不快速塌成单一 expert
- `selected_forecast_orig_mse_mean` 相比初始 SFT 基线下降
- `orig_mse_mean` 相比初始 SFT 基线下降
- `refinement_delta_orig_mse_mean >= 0`

### 2.3 论文对齐目标
- 保留 `<think>...</think><answer>...</answer>` 外壳
- 保留 3-turn 流程语义
- 保留 diagnostic → routing → refinement 的任务分解
- 不删除当前 tool-augmented 主线，只做稳定化修复

---

## 3. 总体实施顺序

按以下顺序执行，**每一阶段必须验收通过再进入下一阶段**：

1. 阶段 0：数据与 expert 审计
2. 阶段 1：最小必要代码修复
3. 阶段 2：重建高质量 SFT 数据集
4. 阶段 3：SFT 两段式训练与验收
5. 阶段 4：RL 第一轮稳定化训练与验收
6. 阶段 5：如果仍无法学好，再做 Turn 3 最小结构增强

---

# 4. 阶段 0：数据与 expert 审计

## 4.1 审计目标
在进入任何训练修改之前，必须先确认以下问题：

1. expert 路由任务是否真的有学习价值；
2. train / val / test 是否存在 source overlap；
3. reference teacher 标签是否稳定、合法；
4. refine 样本是否严重失衡；
5. teacher 使用的 expert 与 runtime 使用的 expert 是否一致。

## 4.2 审计项 A：expert 路由是否有价值
基于已有 benchmark 结果检查：

- `winner_distribution`
- `best_vs_second_gap`
- `val / train 上 expert winner 是否并非完全单一`

### 验收标准
- 没有单 expert 占比 > 80%
- `best_vs_second_gap` 虽然可能偏小，但至少应存在可区分样本
- 如果 route 任务几乎退化成单 expert，则后续 route 监督要降权，不能继续强追“多专家学习”叙事

## 4.3 审计项 B：split 泄漏检查
检查 train / val / test 的 `source_sample_index` 或其等价 id 是否重叠。

### 验收标准
- train / val / test source id 交集必须为 0

## 4.4 审计项 C：teacher 标签合法性
对 curated jsonl 检查：

- `reference_teacher_model` 非空率
- `reference_teacher_error` 是否有限
- `reference_teacher_prediction_text` 是否严格长度 = 96
- `selected_prediction_model == reference_teacher_model` 的比例

### 验收标准
- teacher 缺失率 = 0
- teacher 预测长度合法率 = 100%
- `selected_prediction_model_reference_teacher_agreement_ratio >= 0.98`

## 4.5 审计项 D：route 标签置信度分布
检查：

- `routing_confidence_tier_distribution`
- `reference_teacher_model_distribution`

### 验收标准
- `mid + high` routing 样本比例尽量 >= 25%
- 如果 `mid + high` 很少，说明 route 标签噪声大，后续 route-only SFT 和 RL 都不能重压低置信样本

## 4.6 审计项 E：refine 样本失衡
检查：

- `train_turn3_target_type_distribution`
- `train_refine_ops_signature_distribution`

### 验收标准
- `local_refine` 样本比例 >= 30%
- 不能几乎全是 `validated_keep`

## 4.7 审计项 F：expert 版本一致性
核对以下三处是否使用同一批 expert：

1. teacher 数据集构造时调用的 expert
2. runtime `predict_time_series` 调用的 expert
3. benchmark / report 中评估的 expert

### 验收标准
- 三者使用同一批 checkpoint / 版本
- 必须记录到 `provenance.json` 或单独的审计报告中

## 4.8 阶段 0 交付物
生成一份 `artifacts/reports/v15_data_audit.md`，至少包含：

- split overlap 检查结果
- teacher 标签合法率
- route 置信度分布
- refine 样本分布
- expert 版本一致性说明
- 是否建议进入阶段 1

---

# 5. 阶段 1：最小必要代码修复

本阶段只做**低风险高收益**的改动，不推翻框架。

---

## 5.1 修改项 A：统一 Turn 3 协议（保留论文外壳）

### 问题
当前存在：

- SFT 默认使用 `paper_strict`，Turn 3 输出完整 forecast 文本；
- runtime refinement 却要求 `decision=<name>`；
- 导致 SFT 教的是一种任务，runtime/RL 又要求另一种任务。

### 修改目标
采用以下折中方案：

- **保留 `<think>...</think><answer>...</answer>` 外层结构**；
- **把 `<answer>` 内部统一成一行结构化 decision**；
- 从而兼顾论文对齐与工程稳定性。

### 修改文件
`recipe/time_series_forecast/build_etth1_sft_dataset.py`

### 修改内容
将：

```python
DEFAULT_TURN3_TARGET_MODE = TURN3_TARGET_MODE_PAPER_STRICT
```

改为：

```python
DEFAULT_TURN3_TARGET_MODE = TURN3_TARGET_MODE_ENGINEERING_REFINE
```

同时**不要**移除 Turn 3 的 XML 外壳（保留 `paper_think_answer_xml` 协议风格）。

### 验收标准
- 新生成的 Turn 3 target 全部使用结构化 decision
- 但外层仍保持 `<think>` / `<answer>` 形式
- `turn3_protocol_valid_ratio = 1.0`

---

## 5.2 修改项 B：routing prompt 去掉答案泄漏

### 问题
当前 routing prompt 中存在过强 heuristic 脚手架，例如：

- `Routing Decision Guide`
- `expert_support_signals`
- 直接提示某种模式应选哪个 expert

这会使模型学成“按 prompt 模板选模型”，而不是根据 state 学 policy。

### 修改目标
改成：

- **弱引导，强证据**；
- 保留工具输出的原始统计与状态摘要；
- 删除直接给 expert 结论的文本。

### 修改文件
`recipe/time_series_forecast/prompts.py`

### 必删内容
- `ROUTING_REASON_GUIDE`
- `expert_support_signals`
- `_routing_support_codes_from_payload()` 的接入输出

### 保留内容
- `observed_tools`
- 工具原始字段
- analysis history
- 中性约束：只能从 `patchtst, itransformer, arima, chronos2` 中选一个

### 推荐改法
将 `build_routing_evidence_card()` 改为只生成：

- 已观测工具列表
- 每个工具对应的关键数值字段
- 缺失工具组

不要在 evidence card 中出现“推荐模型”或“support signal”。

### 验收标准
- 新 routing prompt 中不再出现 expert-support 结论语句
- 仍保留足够统计证据
- 诊断 state 信息量不低于原先的 70%

---

## 5.3 修改项 C：降低 KL，并打开小 entropy

### 问题
当前 RL 过于保守：

- `KL=0.04`
- `entropy=0`

这会导致：

- rollout 多样性不足
- route / refine 很快塌成保守行为
- advantage 稀疏

### 修改目标
先做保守但有效的探索增强：

- `KL = 0.01`
- `entropy_coeff = 0.001`
- `temperature = 0.9`

### 修改文件 1
`examples/time_series_forecast/configs/etth1_ot_qwen3_gpu012.sh`

将：

```bash
export RL_KL_LOSS_COEF="${RL_KL_LOSS_COEF:-0.04}"
export RL_TEMPERATURE="${RL_TEMPERATURE:-1.0}"
```

改为：

```bash
export RL_KL_LOSS_COEF="${RL_KL_LOSS_COEF:-0.01}"
export RL_ENTROPY_COEFF="${RL_ENTROPY_COEFF:-0.001}"
export RL_TEMPERATURE="${RL_TEMPERATURE:-0.9}"
```

### 修改文件 2
`examples/time_series_forecast/run_qwen3-1.7B.sh`

把硬编码：

```bash
"actor_rollout_ref.actor.entropy_coeff=0"
```

改为读取变量：

```bash
require_env RL_ENTROPY_COEFF "Set RL_ENTROPY_COEFF in PROFILE_PATH or override it."
```

并在启动参数中使用：

```bash
"actor_rollout_ref.actor.entropy_coeff=$RL_ENTROPY_COEFF"
```

### 验收标准
- RL 配置最终生效值：`KL=0.01, entropy=0.001, temp=0.9`
- rollout 日志中能看到 entropy_coeff 非 0

---

## 5.4 修改项 D：减弱 Turn 3 keep 偏置

### 问题
当前 refine prompt 对 `keep_baseline` 过于友好，导致模型容易学成：

- 一不确定就 keep
- route 错了也 keep
- refine 几乎不工作

### 修改文件
`recipe/time_series_forecast/prompts.py`

### 修改内容
在 `build_refinement_evidence_card()` 中，将：

```python
decision_options = ["keep_baseline"]
...
decision_options.append(adjustment)
```

改为：

```python
decision_options = []
...
decision_options.append(adjustment)
if keep_baseline_allowed:
    decision_options.append("keep_baseline")
```

同时把 prompt 中：

```text
`keep_baseline` is always allowed when the selected forecast already matches the evidence card.
```

改为：

```text
Choose keep_baseline only when no listed adjustment is directly supported by the evidence.
```

### 验收标准
- `keep_baseline` 不再总是排在首位
- refinement prompt 中不再有强鼓励 keep 的文案

# 6. 阶段 2：重建更干净的 SFT 数据集

本阶段目标：

- 用统一协议和更干净的标签重建数据；
- 把 route 高置信样本和 local_refine 样本强化出来；
- 建立新的 v15 数据目录，避免继续污染旧版本数据。

---

## 6.1 构造高质量 curated teacher 数据

### 输出目录
建议新建：

```text
dataset/ett_sft_etth1_runtime_teacher200_refteacher_v15
```

### 推荐命令
```bash
python -m recipe.time_series_forecast.build_etth1_high_quality_sft \
  --train-jsonl dataset/ett_rl_etth1_paper_same2/train.jsonl \
  --val-jsonl dataset/ett_rl_etth1_paper_same2/val.jsonl \
  --test-jsonl dataset/ett_rl_etth1_paper_same2/test.jsonl \
  --output-dir dataset/ett_sft_etth1_runtime_teacher200_refteacher_v15 \
  --models patchtst,chronos2,itransformer,arima \
  --predictor-mode local \
  --resume-teacher-eval \
  --train-target-samples 200 \
  --val-target-samples 64 \
  --test-target-samples 128 \
  --train-candidate-samples 800 \
  --val-candidate-samples 256 \
  --test-candidate-samples 256 \
  --train-min-local-refine-ratio 0.30 \
  --max-turn3-annotation-error-count 0 \
  --max-turn3-annotation-error-ratio 0.0
```

### 设计理由
- 用较大候选池筛出更高质量 200 个 train 样本；
- 保证 local refine 比例；
- Turn 3 标注错误直接视为构造失败。

---

## 6.2 将 curated jsonl 转为 stepwise parquet

### 输出目录
```text
dataset/ett_sft_etth1_runtime_teacher200_refteacher_v15_stepwise
```

### 推荐命令
```bash
python -m recipe.time_series_forecast.build_etth1_sft_dataset \
  --train-jsonl dataset/ett_sft_etth1_runtime_teacher200_refteacher_v15/train_curated.jsonl \
  --val-jsonl dataset/ett_sft_etth1_runtime_teacher200_refteacher_v15/val_curated.jsonl \
  --test-jsonl dataset/ett_sft_etth1_runtime_teacher200_refteacher_v15/test_curated.jsonl \
  --output-dir dataset/ett_sft_etth1_runtime_teacher200_refteacher_v15_stepwise \
  --turn3-target-mode engineering_refine \
  --routing-label-source reference_teacher \
  --sft-stage-mode full \
  --train-turn3-rebalance-mode oversample_local_refine \
  --train-min-local-refine-ratio 0.30 \
  --train-routing-confidence-min-tier mid \
  --train-high-confidence-routing-repeat-factor 2 \
  --train-local-refine-refinement-repeat-factor 2 \
  --train-stage-repeat-factors '{"diagnostic":1,"routing":2,"refinement":2}' \
  --balance-train-routing-models
```

### 参数设计理由
- `routing_label_source=reference_teacher`：route 标签不再使用 heuristic
- `turn3_target_mode=engineering_refine`：统一 Turn 3 协议
- `train-routing-confidence-min-tier=mid`：低置信 route 样本降权
- `oversample_local_refine`：防止 keep 淹没 refine
- `routing/refinement` 重复因子提高：强化真正要学的阶段

---

## 6.3 阶段 2 验收门槛
以下条件必须全部满足：

- `turn3_protocol_valid_ratio = 1.0`
- train / val / test split 无 overlap
- `selected_prediction_model_reference_teacher_agreement_ratio >= 0.98`
- `local_refine_ratio >= 0.30`
- `mid+high route ratio >= 0.25`（尽量）
- train routing model distribution 没有单 expert > 80%

如果不满足，则**禁止进入 SFT 训练阶段**。

# 7. 阶段 3：SFT 两段式训练与验收

本阶段的核心思想：

- **先学 route，再学 full stepwise**；
- 让 route 学习不被 Turn 3 干扰；
- 先确认 parser / 协议 / 路由是否稳，再谈 RL。

---

## 7.1 SFT-A：route-only warm-up

### 输出目录
```text
dataset/ett_sft_etth1_runtime_teacher200_refteacher_v15_routing_only
```

### 推荐命令
```bash
python -m recipe.time_series_forecast.build_etth1_sft_dataset \
  --train-jsonl dataset/ett_sft_etth1_runtime_teacher200_refteacher_v15/train_curated.jsonl \
  --val-jsonl dataset/ett_sft_etth1_runtime_teacher200_refteacher_v15/val_curated.jsonl \
  --test-jsonl dataset/ett_sft_etth1_runtime_teacher200_refteacher_v15/test_curated.jsonl \
  --output-dir dataset/ett_sft_etth1_runtime_teacher200_refteacher_v15_routing_only \
  --turn3-target-mode engineering_refine \
  --routing-label-source reference_teacher \
  --sft-stage-mode routing_only \
  --train-routing-confidence-min-tier mid \
  --train-high-confidence-routing-repeat-factor 2 \
  --balance-train-routing-models
```

### 训练建议
- epoch：0.5 ~ 1
- 目标：先把 routing 学进来

---

## 7.2 SFT-B：full stepwise
使用阶段 2 构造出的 `..._v15_stepwise` 数据，再做 1 epoch full SFT。

### 目标
- 在 route 学稳后再加入 refinement
- 保证三阶段协议可执行

---

## 7.3 SFT 阶段测试项

### route 侧
- 使用 `probe_routing_policy.py` 检查：
  - 高置信 val 样本上的 route agreement
  - `selected_model_distribution`

### refine 侧
- refinement parser 成功率
- `invalid_decision_shape_ratio`
- `refinement_changed_ratio`

### 最终预测侧
- `selected_forecast_orig_mse_mean`
- `orig_mse_mean`
- `refinement_delta_orig_mse_mean`

---

## 7.4 SFT 阶段验收门槛
必须全部满足：

- route parser / refine parser 成功率 > 99.5%
- `invalid_decision_shape_ratio < 1%`
- 高置信 val 样本 route agreement >= 70%
- `refinement_changed_ratio` 处于 5% ~ 40% 区间
- `orig_mse_mean <= selected_forecast_orig_mse_mean + 很小误差`

### SFT 不通过的判定
如果出现以下任一情况，则禁止进入 RL：

- refine 几乎全 rejected
- route agreement 很低
- route 快速塌成单 expert
- `orig_mse_mean` 比 selected baseline 更差

此时应回退到：

- 数据集检查
- prompt 检查
- parser / protocol 检查

# 8. 阶段 4：RL 第一轮稳定化训练与验收

本阶段不先大改 reward，只做保守但必要的探索增强和协议稳定化。

---

## 8.1 RL 第一轮配置

### 保留
- 3-turn agent flow
- delayed final reward 主体
- 现有 GRPO 主线

### 修改
- `RL_KL_LOSS_COEF=0.01`
- `RL_ENTROPY_COEFF=0.001`
- `RL_TEMPERATURE=0.9`
- `RL_VAL_TEMPERATURE=0.0`

### 暂不修改
- 不重写 reward 成 full dense reward
- 不改成 2-turn
- 不移除 tool-augmented 主线

---

## 8.2 RL 启动建议
推荐基于新的 full SFT checkpoint 启动：

```bash
RL_KL_LOSS_COEF=0.01 \
RL_ENTROPY_COEFF=0.001 \
RL_TEMPERATURE=0.9 \
RL_VAL_TEMPERATURE=0.0 \
RL_MODEL_PATH=/path/to/new_sft_checkpoint/huggingface \
PROFILE_PATH=examples/time_series_forecast/configs/etth1_ot_qwen3_gpu012.sh \
RUN_MODE=train \
bash examples/time_series_forecast/run_qwen3-1.7B.sh
```

---

## 8.3 RL 第一轮必须监控的指标
不能只看 reward，必须同时看：

1. `reward_negative_one_ratio`
2. `format_failure_ratio`
3. `selected_model_distribution`
4. `selected_forecast_orig_mse_mean`
5. `orig_mse_mean`
6. `refinement_delta_orig_mse_mean`

### 解释
- 如果协议指标变好，但 `selected_forecast_orig_mse_mean` 不动：说明 route 没学会
- 如果 route 指标变好，但 `orig_mse_mean` 不动：说明 refine 没学会
- 如果 route 迅速塌成单 model：说明 KL 仍过大或状态证据不足

---

## 8.4 RL 阶段验收门槛

### 前 50~100 steps
- `reward_negative_one_ratio < 5%`
- `format_failure_ratio < 2%`
- 单 model 占比不超过 85%（除非 teacher 本身已接近单 expert）

### 一个验证周期后
- `orig_mse_mean <= SFT checkpoint 的 orig_mse_mean`
- `orig_mse_mean <= selected_forecast_orig_mse_mean`
- `refinement_delta_orig_mse_mean >= 0`

### 两个验证周期后
- `orig_mse_mean` 相比 SFT 基线改善 2% ~ 5%

### 如果不达标
如果协议稳定了但核心预测指标不动，应进入阶段 5，不再继续长时间硬跑当前结构。

---

# 9. 阶段 5：最小结构增强（仅在必要时）

只有在以下条件成立时才进入本阶段：

- 数据已重建
- Turn 3 协议已统一
- route prompt 泄漏已去掉
- KL / entropy 已调整
- SFT / RL 协议稳定
- 但 `selected_forecast_orig_mse_mean` 能改善、`orig_mse_mean` 仍不改善

这说明：

- route 可能开始学了
- 但 refine 补救能力太弱

---

## 9.1 最小增强思路
不切到 full v2，只增强 Turn 3 的补救能力。

### 当前问题
Turn 3 基本只能在：

- routed expert baseline
- 3 个局部 edit
- keep

之间做选择，无法真正“救回 route 错误”。

### 增强方案
Turn 3 仍保持 `decision=<name>`，但 decision 候选扩展为：

- routed expert baseline
- routed expert 的局部 refine
- **一个 best alternative expert baseline**
- **一个 best alternative expert 的局部 refine**

### 优点
- 保留 3-turn 结构
- 不变成 full rerank 系统
- 但让 Turn 3 具备真正的补救能力

---

## 9.2 阶段 5 验收门槛
- `refinement_changed_ratio` 上升，但不过高（避免乱改）
- `final_vs_selected_mse_mean` 明显改善
- `orig_mse_mean` 相比阶段 4 明显改善
- route 不必完美，但 final 输出质量应明显提升

# 12. 一句话执行总结

这套方案的核心不是“调一个参数让模型 magically 变好”，而是建立一个**逐关验收的训练闭环**：

1. 先确认数据没教歪；
2. 再修协议不一致和 prompt 泄漏；
3. 再用高质量 route/refine 样本训稳 SFT；
4. 最后才用较温和的 RL 去学更好的 policy；
5. 如果 refine 仍然救不回 route，再做最小结构增强。

只有这样，才能真正判断模型为什么没学稳、没学好，并让它逐步进入稳定学习状态。
